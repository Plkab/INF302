Communication via SPI

Protocole

Le bus SPI (Serial Peripheral Interface) est un bus synchrone qui utilise seulement 4 fils pour la communication :

  • SCLK (Serial Clock) : horloge générée par le maître.
  • MOSI (Master Out Slave In) : données du maître vers l'esclave.
  • MISO (Master In Slave Out) : données de l'esclave vers le maître.
  • CE ou SS (Chip Enable / Slave Select) : permet de sélectionner l'esclave actif à l'état bas en général.

Le maître et l'esclave possèdent chacun un registre à décalage de 8 bits. Lorsque le maître génère 8 impulsions d'horloge, les octets sont échangés simultanément entre les deux registres. La communication est full-duplex (émission et réception simultanées).

Le maître et l’esclave doivent s’accorder sur la polarité (CPOL) et la phase (CPHA) de l’horloge.

  • CPOL (Clock Polarity) : définit le niveau au repos de l'horloge (0 = bas, 1 = haut).
  • CPHA (Clock Phase) : définit sur quel front l'échantillonnage des données a lieu (0 = premier front, 1 = second front). Les combinaisons donnent 4 modes (0, 1, 2, 3). Par exemple, le mode 0 (CPOL=0, CPHA=0) lit les données sur le front montant et les change sur le front descendant.

Configuration du SPI

Nous allons configurer le STM32F401 comme maitre. Il integre 3 modules SPI (SPI1, SPI2, SPI3).

Nous utiliserons SPI1 avec les broches suivantes :

  • PA5 → SCK
  • PA7 → MOSI
  • PA4 → CS (géré par logiciel, en sortie GPIO)

Les registres importants pour la programmation SPI sont :

  • SPI_CR1 : contrôle principal (baud rate, CPOL, CPHA, maître/esclave, taille des données, activation).
  • SPI_CR2 : configuration supplémentaire (interruptions, format trame).
  • SPI_SR : registre d’état (TXE, RXNE, BSY).
  • SPI_DR : registre de données (écriture pour émettre, lecture pour recevoir).

Config typique :

  • Activer l'horloge du périphérique SPI1 via RCC_AHB1ENR.
  • Activer l’horloge du port GPIOA via RCC_AHB1ENR.
  • Configurer PA5 = SCK, PA7 = MOSI en mode alternate function (AF5 pour SPI1)
  • Configurer la broche GPIO PA4 en sortiepush-pull pour gérer le CS.
  • Configurer le registre SPI_CR1
  • MSTR = 1 (mode maître).
  • BR[2:0] : diviseur de l'horloge (ex. 011 pour diviser par 16 → fPCLK/16, si APB2 = 84 MHz → 5,25 MHz).
  • CPOL et CPHA selon l'esclave utilisé.
  • DFF = 0 pour des trames de 8 bits ou DFF = 1 pour une trame de 16 bits (le MCP4921 attend 16 bits)
  • LSBFIRST = 0 (evoie le MSB en premier)
  • SPE = 1 pour activer le module.
  • Pour faire une transmission, il faut attendre que le flag TXE (Transmit Buffer Empty) soit à 1 dans SPI_SR, puis abaisser le SS (actif à 0). Ensuite écrire l'octet dans SPI_DR, puis attende que le flag BSY (Busy) soit à 0 dans SPI_SR. Et enfin remontez le SS.
void SPI1_Init(void)
{
    RCC->AHB1ENR |= (1 << 0);      // GPIOA
    RCC->APB2ENR |= (1 << 12);     // SPI1

    // PA5 (SCK) et PA7 (MOSI) en Alternate Function
    GPIOA->MODER &= ~((3 << 10) | (3 << 14));
    GPIOA->MODER |=  ((2 << 10) | (2 << 14));
    GPIOA->AFR[0] &= ~((0xF << 20) | (0xF << 28));
    GPIOA->AFR[0] |=  ((5 << 20)  | (5 << 28));

    // PA4 en sortie pour CS
    GPIOA->MODER &= ~(3 << 8);
    GPIOA->MODER |=  (1 << 8);
    GPIOA->BSRR = (1 << 4);        // CS inactif

    // Configuration SPI
    SPI1->CR1 = 0;
    SPI1->CR1 |= (1 << 2);         // MSTR
    SPI1->CR1 |= (1 << 8) | (1 << 9); // SSM + SSI
    SPI1->CR1 |= (1 << 11);        // DFF (trame 16 bits)
    SPI1->CR1 |= (3 << 3);         // BR = 011 (5.25 MHz si APB2=84MHz)
    SPI1->CR1 |= (1 << 6);         // SPE
}

Interface SPI entre STM32F401 et DAC MCP4921

Le MCP4921 est un convertisseur numérique-analogique (DAC) de 12 bits. Il permet de transformer une valeur numérique envoyée par un microcontrôleur en une tension analogique. Il est très utilisé dans les systèmes embarqués pour :

  • Génération de signaux sinusoïdaux, triangulaires ou carrés
  • Synthèse sonore (DDS, FM, modulation)
  • Commande analogique de variateurs et moteurs
  • Génération de tensions de référence
  • Commande analogique de moteurs
  • Instrumentation et systèmes de mesure

Le MCP4921 communique exclusivement par le protocole SPI, ce qui le rend particulièrement adapté aux microcontrôleurs STM32. Il reçoit un mot de 16 bits organisé comme suit :

Bit Nom Description
15 A/B 0 = DAC A (unique)
14 BUF 1 = tampon activé (sortie à faible impédance)
13 GA 0 = gain 1x (Vref), 1 = gain 2x
12 SHDN 1 = sortie active, 0 = mode veille
11-0 D[11:0] Valeur numérique 12 bits

Le MCP4921 fonctionne avec les modes suivants (voir sa datasheet) :

  • Mode 0, 0 (CPOL = 0, CPHA = 0) : l’horloge est à 0 au repos, les données sont lues sur le front montant (premier front).
  • Mode 1, 1 (CPOL = 1, CPHA = 1) : l’horloge est à 1 au repos, les données sont lues sur le front descendant.

Nous choisirons le mode 0,0 (le plus courant) :

  • A/B = 0
  • BUF = 1
  • GA = 0 (gain 1x)
  • SHDN = 1

Le mot de commande devient alors : 0b0100 1xxx xxxx xxxx soit en hexadécimal 0x3000 pour une valeur de sortie nulle, et on ajoute les 12 bits de données dans les bits 11..0.

Exemple : pour une valeur 12 bits val (0 à 4095), le mot 16 bits = 0x3000 | val.

Procédure d’émission vers le MCP4921

  • Mettre la broche CS à l’état bas.Attendre que le tampon d’émission soit vide (TXE = 1 dans SPI_SR).
  • Écrire le mot de 16 bits dans SPI_DR.
  • Attendre la fin de la transmission (BSY = 0 dans SPI_SR).
  • Remettre CS à l’état haut.

Le MCP4921 met à jour sa sortie analogique sur le front montant de CS. Il est donc impératif de respecter cette séquence.

Calcul de la tension

La formule est : V_OUT = V_REF × (Valeur_DAC / 4096)

Exemple : * V_REF = 3.3V * Valeur DAC = 2048

Résultat : V_OUT = 3.3 × (2048 / 4096) = 1.65V

/*==============================================================
    Envoi d'une valeur au DAC MCP4921
==============================================================*/
void DAC_Write(uint16_t value)
{
    uint16_t commande = 0x3000 | (value & 0x0FFF);

    // Attendre que le buffer d'émission soit vide
    while (!(SPI1->SR & (1 << 1)));

    // CS = 0 (sélection du DAC)
    GPIOA->BSRR = (1 << 20);

    // Envoi des 16 bits
    SPI1->DR = commande;

    // Attendre la fin de la transmission
    while (!(SPI1->SR & (1 << 1)));
    while (SPI1->SR & (1 << 7));

    // CS = 1 (désélection)
    GPIOA->BSRR = (1 << 4);
}

/*==============================================================
    Temporisation logicielle (approximative)
    Ajuster la constante pour 1 ms selon la fréquence CPU
==============================================================*/
void delay_ms(uint32_t ms)
{
    volatile uint32_t i;
    while (ms--)
    {
        for (i = 0; i < 21000; i++); // ~1 ms à 84 MHz
    }
}

Le code qu'on a fait precedement est strictement bloquant (polling). Regardez ces deux lignes :

while (!(SPI1->SR & SPI_SR_TXE)) {}  // Attend que le buffer d'émission soit vide 
while (SPI1->SR & SPI_SR_BSY) {}      // Attend la fin de la transmission SPI

Tant que la condition n’est pas remplie, le processeur reste verrouillé sur cette ligne (il tourne à vide). Il ne peut ni traiter une autre tâche, ni répondre à une interruption, ni passer en mode basse consommation. C’est ce qu’on appelle du polling actif ou busy-waiting.

Il faut noter que dans un système embarqué "produit", on évite le busy-waiting dès qu’on a plus de 2 ou 3 périphériques à gérer.

Générateur de Sinusoïde Non-Bloquant

L’approche précédente utilisait des boucles d’attente (while) qui bloquent le CPU. Pour un système embarqué réel, il est impératif de libérer le processeur afin qu’il puisse traiter d’autres tâches (affichage, lecture de capteurs, communication, etc.).

La solution consiste à utiliser un timer matériel pour générer une interruption à la fréquence d’échantillonnage souhaitée, et à placer l’envoi des données au DAC dans cette interruption. Ainsi, le CPU n’est sollicité que très brièvement à chaque échantillon, et le reste du temps, il peut exécuter la boucle principale ou d’autres tâches.

Principe

  • Table de sinus : Une table de 256 points est stockée en mémoire Flash.
  • Déclenchement périodique : Un timer (par exemple TIM2) est configuré pour générer une interruption périodique.
  • Routine d'interruption (ISR) : Dans l’interruption du timer, on effectue les actions suivantes :
  • Lecture de la valeur correspondant à l’index courant dans la table de sinus.
  • Envoi de la valeur au DAC via SPI (avec un polling court mais acceptable car le temps de transmission est inférieur à 5 µs).
  • Incrémentation de l’index.
  • Contrôle de la fréquence : La fréquence de la sinusoïde est déterminée par la période du timer. Pour modifier la fréquence, on ajuste le registre ARR (Auto-Reload Register) du timer en cours de fonctionnement (par exemple, grâce à un potentiomètre lu par l’ADC).
  • Gestion de l'ADC : L’ADC est lu dans la boucle principale (polling non bloquant via un délai basé sur le tick système) ou par interruption pour une lecture périodique.

    }

/*==============================================================
   Initialisation du SPI1 (identique à avant)
==============================================================*/
void SPI1_Init(void)
{
    RCC->AHB1ENR |= (1 << 0);      // GPIOA
    RCC->APB2ENR |= (1 << 12);     // SPI1

    GPIOA->MODER &= ~((3 << 10) | (3 << 14));
    GPIOA->MODER |=  ((2 << 10) | (2 << 14));
    GPIOA->AFR[0] &= ~((0xF << 20) | (0xF << 28));
    GPIOA->AFR[0] |=  ((5 << 20)  | (5 << 28));

    GPIOA->MODER &= ~(3 << 8);
    GPIOA->MODER |=  (1 << 8);
    GPIOA->BSRR = (1 << 4);        // CS inactif

    SPI1->CR1 = 0;
    SPI1->CR1 |= (1 << 2);         // MSTR
    SPI1->CR1 |= (1 << 8) | (1 << 9); // SSM + SSI
    SPI1->CR1 |= (1 << 11);        // DFF
    SPI1->CR1 |= (3 << 3);         // BR = 011 (5.25 MHz)
    SPI1->CR1 |= (1 << 6);         // SPE
}

/*==============================================================
   Envoi d'une valeur au DAC (polling court)
   Appelé depuis l'interruption TIM2
==============================================================*/
void DAC_Write(uint16_t value)
{
    uint16_t commande = 0x3000 | (value & 0x0FFF);

    while (!(SPI1->SR & (1 << 1)));   // Attente TXE (courte)
    GPIOA->BSRR = (1 << 20);          // CS = 0
    SPI1->DR = commande;
    while (!(SPI1->SR & (1 << 1)));   // Attente TXE (courte)
    while (SPI1->SR & (1 << 7));      // Attente BSY (courte)
    GPIOA->BSRR = (1 << 4);           // CS = 1
}

/*==============================================================
   Initialisation du Timer2 (interruption périodique)
==============================================================*/
void TIM2_Init(void)
{
    RCC->APB1ENR |= (1 << 0);        // Activer TIM2

    TIM2->PSC = 0;                   // Pas de prescaler (16 MHz)
    TIM2->ARR = ARR_VALUE;           // Valeur initiale (1 kHz)
    TIM2->DIER |= (1 << 0);          // Activer l'interruption sur Update
    TIM2->CR1 |= (1 << 0);           // Démarrer le timer

    NVIC_EnableIRQ(TIM2_IRQn);
    NVIC_SetPriority(TIM2_IRQn, 0);  // Priorité haute
}

/*==============================================================
   Interruption TIM2 – cœur du générateur
==============================================================*/
void TIM2_IRQHandler(void)
{
    if (TIM2->SR & (1 << 0))         // Vérifier le flag Update
    {
        TIM2->SR &= ~(1 << 0);       // Effacer le flag

        // Envoyer l'échantillon courant
        DAC_Write(sine_table[sinus_index]);

        // Incrémenter l'index (boucle modulo 256)
        sinus_index++;
        if (sinus_index >= TABLE_SIZE)
            sinus_index = 0;
    }

Gestion du Timer TIM2 & Fréquence de Sortie

  • Fréquence d'échantillonnage : Le TIM2 est configuré pour générer une interruption périodique (initialement à 1 kHz).
  • Routine d'interruption (ISR) : À chaque appel de l'interruption, le système exécute deux actions :
  • Envoi de la valeur courante de la table de sinus au DAC.
  • Incrémentation de l'index de la table.
  • Calcul de la fréquence (\(f_{out}\)) : Avec une table de 256 points, la fréquence de la sinusoïde générée est définie par la formule : $\(f_{out} = \frac{f_{ech}}{256}\)$
  • Contrôle dynamique : En modifiant le registre ARR (Auto-Reload Register) du timer depuis la boucle principale, on fait varier la fréquence d'échantillonnage (\(f_{ech}\)), ajustant ainsi directement la fréquence de sortie (\(f_{out}\)).

Interface Potentiomètre & Polling SPI

  • Réglage par potentiomètre : L’ADC est lu en mode polling rapide dans la boucle principale. La valeur brute acquise (0 à 4095) est convertie en une fréquence cible (1 à 50 Hz) pour recalculer et mettre à jour le registre ARR en temps réel, sans aucun blocage CPU.
  • Polling SPI dans l'ISR : L'interruption TIM2 utilise des boucles while courtes pour attendre les drapeaux TXE (Transmit Buffer Empty) et BSY (Busy).
  • Impact temporel : Cette attente reste acceptable car la transmission d'une trame de 16 bits à 5,25 MHz ne dure qu’environ 3 µs. Bien que les interruptions de priorité inférieure soient masquées pendant ce court instant, l'impact sur le système reste négligeable.

Organisation de la Boucle Principale

  • Libération du processeur : La boucle principale (while(1)) est totalement déchargée et libre d'exécuter d'autres tâches (calculs complexes, affichage LCD, communication UART).
  • Exécution fluide : Elle ne contient aucune fonction d'attente bloquante. La lecture périodique de l'ADC est cadencée sans jamais ralentir le reste des traitements applicatifs.

Générateur de Signaux avec DDS (Direct Digital Synthesis)

Nous allons adapter notre générateur de sinusoïde pour utiliser la technique DDS (Direct Digital Synthesis). Cela nous permettra de contrôler la fréquence avec une très grande précision et de générer des formes d'onde complexes (sinus, carré, dent de scie, etc.) en modifiant simplement la table ou l’algorithme de génération.

Principe du DDS

  • Accumulateur de phase : Un registre de phase (phase_acc) est incrémenté à chaque échantillon d’une valeur fixe appelée le pas (phase_step).
  • Résolution de l'accumulateur : Le système utilise ici un accumulateur codé sur \(N = 32\) bits.
  • Extraction de l'index : L’index permettant de lire la table d'onde est extrait des bits de poids fort de phase_acc. Avec une table de 256 points (\(2^8\)), on utilise les 8 bits de poids fort (obtenus par un décalage binaire de 24 bits : phase_acc >> 24).

Formules Mathématiques

1. Fréquence de sortie (\(f_{out}\))

La fréquence du signal généré dépend du pas d'incrémentation, de la fréquence d'échantillonnage (\(f_{ech}\)) et de la taille de l'accumulateur (\(2^N\)) :

\[f_{out} = \frac{\text{phase\_step} \times f_{ech}}{2^N}\]

2. Calcul du pas (\(\text{phase\_step}\))

Pour configurer le générateur sur une fréquence cible (\(f_{desiree}\)), la boucle principale doit calculer le pas requis selon la formule suivante :

\[\text{phase\_step} = \frac{f_{desiree} \times 2^N}{f_{ech}}\]

[!TIP] Avantage majeur du DDS : Le changement de fréquence est instantané et sans rupture de phase. Il suffit de modifier la variable phase_step dans le code, sans jamais toucher à la configuration matérielle du timer (\(f_{ech}\) reste fixe).

void SPI1_Init(void)
{
    RCC->AHB1ENR |= (1 << 0);      // GPIOA
    RCC->APB2ENR |= (1 << 12);     // SPI1

    // PA5 (SCK), PA7 (MOSI) en mode Alternate Function AF5
    GPIOA->MODER &= ~((3 << 10) | (3 << 14));
    GPIOA->MODER |=  ((2 << 10) | (2 << 14));
    GPIOA->AFR[0] &= ~((0xF << 20) | (0xF << 28));
    GPIOA->AFR[0] |=  ((5 << 20)  | (5 << 28));

    // PA4 en sortie pour CS
    GPIOA->MODER &= ~(3 << 8);
    GPIOA->MODER |=  (1 << 8);
    GPIOA->BSRR = (1 << 4);

    // Configuration SPI : maître, 16 bits, 5.25 MHz
    SPI1->CR1 = 0;
    SPI1->CR1 |= (1 << 2);         // MSTR
    SPI1->CR1 |= (1 << 8) | (1 << 9); // SSM+SSI
    SPI1->CR1 |= (1 << 11);        // DFF (16 bits)
    SPI1->CR1 |= (3 << 3);         // BR = 011
    SPI1->CR1 |= (1 << 6);         // SPE
}

{

    while (!(SPI1->SR & (1 << 1)));   // Attendre TXE

void TIM2_Init(void)

    RCC->APB1ENR |= (1 << 0);        // TIM2 ON
    TIM2->PSC = 0;                   // Pas de prescaler (16 MHz)

    TIM2->CR1 |= (1 << 0);           // Démarrage

    ADC1->CR1 = 0;
    ADC1->CR2 |= (1 << 0);           // ADON

  • Calcul de l'index : Les 8 bits de poids fort de l'accumulateur (obtenus par l'opération phase_acc >> 24) fournissent un index compris entre 0 et 255 pour pointer dans la table de sinus.
  • Calcul du pas (\(\text{phase\_step}\)) : Le pas est recalculé dynamiquement lors de chaque changement de fréquence cible via la formule suivante : $\(\text{phase\_step} = \frac{\text{freq} \times 2^{32}}{\text{SAMPLE\_FREQ}}\)$

Avantages de cette méthode

  • Précision extrême : La fréquence de sortie possède une résolution fine définie par : $\(\Delta f = \frac{\text{SAMPLE\_FREQ}}{2^{32}} \approx 0.0023 \text{ Hz}\)$
  • Grande flexibilité : La modification ou le remplacement de la table de conversion (sinus, carré, triangle, etc.) suffit pour générer instantanément n'importe quelle forme d'onde.
  • Excellente stabilité : Le pas d'incrémentation demeure rigoureusement identique entre deux requêtes de changement de fréquence. Le signal généré est donc parfaitement stable et constant.

Générateur de signaux par DDS avec STM32F401 + MCP4921

Ce code implémente un générateur de signaux (sinus, carré, dent de scie, triangle, rampe) avec une synthèse numérique directe (DDS) temps réel, utilisant une interruption TIM2 pour un échantillonnage stable à 10 kHz, une table de sinus de 1024 points, et des protothreads pour les tâches non critiques (ADC, LCD, bouton).

Vue d’ensemble : Une Architecture à Deux Niveaux

Votre système est divisé en deux mondes qui cohabitent harmonieusement :

Niveau Composants Rôle
Temps réel strict TIM2 (DDS) + TIM3 (déclenchement ADC) Gère la génération du signal et l’acquisition du potentiomètre de manière déterministe. Les interruptions y sont prioritaires.
Applicatif / Interface Protothreads (LCD, mise à jour fréquence) Gère l’affichage et les calculs de mise à jour sans jamais bloquer le cœur temps réel.

Cette séparation garantit que la sortie du DAC n’est jamais perturbée par des opérations lentes (comme écrire sur le LCD).

2. ADC en Interruption Déclenché par TIM3 (100 Hz)

Ce qui est fait

  • Le Timer3 est configuré pour compter jusqu’à une valeur donnée, puis générer un signal TRGO (Trigger Output).
  • Ce signal est câblé en dur à l’entrée de déclenchement externe de l'ADC1.
  • À chaque impulsion (100 Hz), l’ADC démarre automatiquement une conversion sur la broche PA0 (potentiomètre).
  • Quand la conversion est terminée, l’interruption ADC_IRQHandler est appelée. Elle lit la donnée (adc_raw) et lève un sémaphore (adc_ready = 1).

Pourquoi c’est bien

  • Pas de polling : Le microcontrôleur ne perd pas de temps à scruter un registre en boucle. C’est le matériel qui le prévient.
  • Régulier : La lecture se fait à 100 Hz (toutes les 10 ms). La fréquence de sortie est mise à jour à cette cadence, évitant les variations trop brusques.
  • Découplage : L’interruption ne fait que stocker la valeur brute. C’est un protothread (task_update_frequency) qui fera le traitement (filtrage, calcul), ce qui rend l’interruption très rapide.

3. DDS dans TIM2 (Priorité 0)

Ce qui est fait

  • Le Timer2 est configuré pour générer une interruption 10 000 fois par seconde (10 kHz).
  • Cette interruption est prioritaire (niveau 0 dans le NVIC) : elle ne peut pas être retardée par d’autres interruptions.
  • Dans cette interruption :
  • Extraction des 10 bits de poids fort de l’accumulateur phase_acc pour obtenir un index dans la table de sinus de 1024 points.
  • Lecture de la valeur dans la table (ou calcul selon le mode : carré, dent de scie, etc.).
  • Envoi de la valeur au DAC via SPI (DAC_Write).
  • Ajout de phase_step à phase_acc.

Pourquoi c’est bien (DDS)

  • Le DDS (Direct Digital Synthesis) permet de changer la fréquence instantanément en modifiant un seul paramètre : phase_step. Aucun changement de timer requis.
  • La table de 1024 points offre une résolution de phase de 0,35° (\(360^\circ / 1024\)), garantissant une sinusoïde très propre.
  • La priorité 0 garantit un jitter temporel inférieur à 1 µs. L'intervalle fixe de 100 µs élimine les micro‑variations de fréquence (phase noise).

4. Protothreads (Tâches UI Non-Bloquantes)

Ce qui est fait

Deux protothreads sont utilisés : * task_update_frequency : Attend le sémaphore adc_ready, filtre la valeur ADC, calcule la fréquence et le nouveau phase_step. * task_lcd : Rafraîchit l’écran LCD toutes les 200 ms ou quand lcd_update est à 1.

Ils utilisent PT_WAIT_UNTIL() pour « dormir » jusqu’à ce qu’un événement se produise.

Pourquoi c’est bien

  • Les protothreads transforment une logique séquentielle en machines à états implicites sans pile de contexte (2 octets de RAM chacun).
  • Le CPU n'est jamais bloqué dans une boucle while vide. Si une interruption TIM2 arrive pendant le rafraîchissement du LCD, elle préempte le code et le signal reste intact.
  • Cela permet d’ajouter facilement d’autres tâches (UART, LED) sans refondre l’architecture.

5. Filtrage Exponentiel (Moyenne Lissée)

Ce qui est fait

Dans task_update_frequency, on applique un filtre numérique :

filtered = (filtered * 7 + adc_raw) >> 3;

C’est une moyenne exponentielle : la nouvelle valeur filtrée est composée de 7/8 de l’ancienne et de 1/8 de la nouvelle valeur brute.

Pourquoi c’est bien

  • Les potentiomètres analogiques sont souvent bruités. Sans filtre, la fréquence « vibrerait » autour de la valeur souhaitée.
  • Ce filtre est extrêmement léger en calcul (un simple décalage binaire >> 3 au lieu d'une division).
  • Il suit les variations lentes (mouvement du doigt) tout en éliminant les fluctuations rapides.

/**
 * @brief Configure l'horloge système sur HSI 16 MHz (sans PLL).
 * 
 * Cela simplifie l'initialisation et évite les réglages complexes.
 * Tous les bus (AHB, APB1, APB2) fonctionnent à 16 MHz.
 */
void clock_init(void) {
    // Activer l'oscillateur interne HSI (High-Speed Internal)
    RCC->CR |= RCC_CR_HSION;
    // Attendre que l'oscillateur soit stable
    while (!(RCC->CR & RCC_CR_HSIRDY));
    // Effacer le registre CFGR (configurations par défaut)
    RCC->CFGR = 0;
    // Sélectionner HSI comme source d'horloge système
    RCC->CFGR |= RCC_CFGR_SW_HSI;
    // Attendre que le basculement soit effectif
    while ((RCC->CFGR & RCC_CFGR_SWS) != RCC_CFGR_SWS_HSI);
}

/**
 * @brief Configure les broches GPIO nécessaires :
 *   - PA4 : CS du DAC (sortie)
 *   - PA5 : SCK (SPI) en mode alternatif AF5
 *   - PA7 : MOSI (SPI) en mode alternatif AF5
 *   - PA0 : entrée analogique pour le potentiomètre
 *   - PC13 : bouton (entrée avec pull-up)
 */
void gpio_init(void) {
    // Activer les horloges des ports GPIOA et GPIOC
    RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN | RCC_AHB1ENR_GPIOCEN;

    /* ---- PA4 : CS du DAC (sortie) ---- */
    GPIOA->MODER &= ~GPIO_MODER_MODER4_Msk; // Effacer les bits de configuration
    GPIOA->MODER |= GPIO_MODER_MODER4_0;    // Mode sortie (0b01)
    GPIOA->BSRR = GPIO_BSRR_BS4;            // Mettre CS à 1 (inactif)

    /* ---- PA5 (SCK) et PA7 (MOSI) : fonction alternative AF5 pour SPI1 ---- */
    // Effacer les bits MODER pour ces broches
    GPIOA->MODER &= ~(GPIO_MODER_MODER5_Msk | GPIO_MODER_MODER7_Msk);
    // Mettre en mode alternatif (0b10)
    GPIOA->MODER |= (GPIO_MODER_MODER5_1 | GPIO_MODER_MODER7_1);
    // Effacer les bits de sélection de fonction alternative (AFRL[5] et AFRL[7])
    GPIOA->AFR[0] &= ~(GPIO_AFRL_AFSEL5_Msk | GPIO_AFRL_AFSEL7_Msk);
    // Affecter la fonction AF5 (valeur 5) pour SPI1
    GPIOA->AFR[0] |= (5 << GPIO_AFRL_AFSEL5_Pos) | (5 << GPIO_AFRL_AFSEL7_Pos);

    /* ---- PA0 : entrée analogique pour l'ADC ---- */
    // Le mode analogique correspond aux deux bits à 1 (0b11)
    GPIOA->MODER |= GPIO_MODER_MODER0;      // Bits 0-1 mis à 1

    /* ---- PC13 : bouton (entrée avec pull-up) ---- */
    GPIOC->MODER &= ~GPIO_MODER_MODER13_Msk; // Entrée (0b00)
    GPIOC->PUPDR |= GPIO_PUPDR_PUPDR13_0;    // Pull-up activé
}

/**
 * @brief Initialise le module SPI1 pour communiquer avec le DAC MCP4921.
 * 
 * Paramètres : maître, trame 16 bits, vitesse 4 MHz, mode 0 (CPOL=0, CPHA=0).
 */
void spi_init(void) {
    // Activer l'horloge du SPI1 (sur APB2)
    RCC->APB2ENR |= RCC_APB2ENR_SPI1EN;

    SPI1->CR1 = 0;                           // Reset complet
    SPI1->CR1 |= SPI_CR1_MSTR;               // Mode maître
    SPI1->CR1 |= SPI_CR1_SSM | SPI_CR1_SSI;  // Gestion du NSS par logiciel (pas de hardware)
    SPI1->CR1 |= SPI_CR1_DFF;                // Trame de 16 bits (MCP4921 attend 16 bits)
    SPI1->CR1 |= SPI_CR1_BR_0;               // BR=001 => fPCLK/4 = 16/4 = 4 MHz
    // CPOL=0 et CPHA=0 par défaut => mode 0
    SPI1->CR1 |= SPI_CR1_SPE;                // Activer le périphérique SPI
}

/**
 * @brief Initialise l'ADC et le Timer3 pour un déclenchement périodique.
 * 
 * Le Timer3 génère un signal TRGO à 100 Hz qui déclenche une conversion ADC.
 * L'interruption de fin de conversion (EOC) stocke la valeur et lève un sémaphore.
 * 
 * Cette méthode est inspirée de l'exemple fonctionnel fourni par l'étudiant.
 */
void adc_timer_init(void) {
    // 1. Activer les horloges de l'ADC1 (APB2) et du Timer3 (APB1)
    RCC->APB2ENR |= RCC_APB2ENR_ADC1EN;
    RCC->APB1ENR |= RCC_APB1ENR_TIM3EN;

    // 2. Configurer PA0 en mode analogique (déjà fait dans gpio_init, mais redondance)
    GPIOA->MODER |= (3 << (0 * 2));        // PA0 analogique

    // 3. Prédiviseur de l'horloge ADC : 16 MHz / 4 = 4 MHz (max 30 MHz)
    ADC->CCR &= ~ADC_CCR_ADCPRE;           // Effacer le champ ADCPRE
    ADC->CCR |= (1 << ADC_CCR_ADCPRE_Pos); // 1 = division par 4

    // 4. Séquenceur : une seule conversion sur le canal 0
    ADC1->SQR3 = 0;                        // Canal 0 (PA0)
    ADC1->SQR1 &= ~ADC_SQR1_L;             // Longueur de séquence = 1

    // 5. Temps d'échantillonnage : 84 cycles d'horloge ADC (précision)
    ADC1->SMPR2 |= (7 << (0 * 3));         // 7 = 84 cycles pour le canal 0

    // 6. Déclenchement externe par le signal TRGO du Timer3
    ADC1->CR2 &= ~ADC_CR2_EXTSEL;          // Effacer le champ EXTSEL
    ADC1->CR2 |= (8 << ADC_CR2_EXTSEL_Pos); // 8 correspond à TIM3 TRGO (voir RM)
    ADC1->CR2 |= (1 << ADC_CR2_EXTEN_Pos); // Front montant sur le déclenchement

    // 7. Activer l'interruption de fin de conversion (EOCIE)
    ADC1->CR1 |= ADC_CR1_EOCIE;

    // 8. Mettre sous tension l'ADC (ADON = 1)
    ADC1->CR2 |= ADC_CR2_ADON;

    // 9. Configurer la priorité et activer l'IRQ ADC dans le NVIC
    NVIC_SetPriority(ADC_IRQn, 1);         // Priorité 1 (0 = plus haute)
    NVIC_EnableIRQ(ADC_IRQn);

    // 10. Configurer le Timer3 pour générer une interruption / TRGO à 100 Hz
    // Horloge TIM3 = 16 MHz (APB1 = 16 MHz car HSI sans prescaler)
    TIM3->PSC = 1600 - 1;                  // Prescaler : division par 1600
    TIM3->ARR = 100 - 1;                   // Auto-reload : 100 ticks => 100 Hz
    TIM3->CR2 |= (2 << TIM_CR2_MMS_Pos);   // Mettre l'événement "Update" sur TRGO (MMS=010)
    TIM3->CR1 |= TIM_CR1_CEN;              // Démarrer le timer
}

/**
 * @brief Initialise le Timer2 pour le cœur DDS (10 kHz).
 * 
 * TIM2 génère une interruption toutes les 100 µs (10 kHz). Dans cette interruption,
 * on calcule l'échantillon suivant et on l'envoie au DAC via SPI.
 * 
 * La priorité de cette interruption est la plus haute (0) pour minimiser le jitter.

void tim2_init(void) {

    RCC->APB1ENR |= RCC_APB1ENR_TIM2EN;
    TIM2->PSC = 0;                         // Pas de prescaler (comptage à 16 MHz)
    TIM2->ARR = 16000000UL / SAMPLE_RATE_HZ - 1; // 1599 (16e6/10000 - 1)
    TIM2->DIER |= TIM_DIER_UIE;            // Activer l'interruption sur mise à jour (update)
    TIM2->CR1 |= TIM_CR1_CEN;              // Démarrer le compteur
    NVIC_EnableIRQ(TIM2_IRQn);
    NVIC_SetPriority(TIM2_IRQn, 0);        // Priorité maximale
}

/**
 * @brief Génère la table de sinus (1024 points) avec des valeurs 12 bits.
 * 
 * Chaque valeur est calculée par : 2048 + 2047 * sin(2π*i/1024).
 * Les valeurs sont arrondies à l'entier le plus proche et saturées à [0,4095].
 */
void generate_sine_table(void) {
    for (int i = 0; i < LUT_SIZE; i++) {
        double val = 2048.0 + 2047.0 * sin(2.0 * M_PI * i / LUT_SIZE);
        sine_lut[i] = (uint16_t)(val + 0.5);  // Arrondi
        // Sécurité : saturer si besoin (normalement inutile)
        if (sine_lut[i] > 4095) sine_lut[i] = 4095;
        if (sine_lut[i] < 0)    sine_lut[i] = 0;
    }
}

/* ============================================================
 * 5. FONCTION DAC (appelée depuis l'interruption TIM2)
 * ============================================================ */

/**
 * @brief Écrit une valeur 12 bits dans le DAC MCP4921 via SPI.
 * 
 * La trame envoyée est : 0x3000 | (value & 0x0FFF).
 * Les bits de commande : A/B=0, BUF=1, GA=0, SHDN=1.
 * La fonction est courte et utilise le polling (attente BSY) car le temps
 * d'émission est très faible (≈ 4 µs à 4 MHz).
 */
void DAC_Write(uint16_t value) {
    if (value > 4095) value = 4095;            // Saturation
    uint16_t command = 0x3000 | (value & 0x0FFF);

    GPIOA->BSRR = GPIO_BSRR_BR4;              // CS = 0 (actif)
    SPI1->DR = command;                       // Envoi des 16 bits
    while (SPI1->SR & SPI_SR_BSY);            // Attendre la fin de la transmission
    GPIOA->BSRR = GPIO_BSRR_BS4;              // CS = 1 (inactif)
}

 * 6. INTERRUPTIONS
 * ============================================================ */

/**
 * @brief Interruption TIM2 (10 kHz) – cœur du DDS.
 * 
 * Cette interruption est prioritaire (niveau 0). Elle calcule l'échantillon
 * suivant à partir de l'accumulateur de phase et le mode sélectionné,
 * puis l'envoie au DAC.
 * 
 * Le switch sur le mode est court ; le temps d'exécution est constant
 * (quelques microsecondes) pour minimiser le jitter.
 */
void TIM2_IRQHandler(void) {
    if (TIM2->SR & TIM_SR_UIF) {
        TIM2->SR &= ~TIM_SR_UIF;               // Effacer le flag

        // Extraction de l'index de la phase (10 bits de poids fort)
        uint16_t idx = (phase_acc >> (PHASE_BITS - INDEX_BITS)) & (LUT_SIZE - 1);

        uint16_t sample;

        // Génération selon le mode actuel
        switch (mode) {
            case MODE_SINE:
                sample = sine_lut[idx];
                break;
            case MODE_SQUARE:
                sample = (idx < (LUT_SIZE/2)) ? 4095 : 0;
                break;
            case MODE_SAWTOOTH:
                sample = (idx * 4095) / (LUT_SIZE - 1);
                break;
            case MODE_TRIANGLE:
                if (idx < LUT_SIZE/2)
                    sample = (idx * 4095) / (LUT_SIZE/2 - 1);
                else
                    sample = ((LUT_SIZE - 1 - idx) * 4095) / (LUT_SIZE/2 - 1);
                break;
            case MODE_RAMP:
                sample = (idx * 4095) / (LUT_SIZE - 1);
                break;
            default:
                sample = 2048;
                break;
        }

        DAC_Write(sample);

        // Mise à jour de l'accumulateur de phase
        phase_acc += phase_step;
    }
}

/**
 * @brief Interruption ADC – déclenchée par TIM3 à 100 Hz.
 * 
 * Lit la valeur convertie, stocke la donnée brute et lève le sémaphore adc_ready.
 * Cette interruption est de priorité 1 (juste derrière TIM2).
 */
void ADC_IRQHandler(void) {
    if (ADC1->SR & ADC_SR_EOC) {
        adc_raw = ADC1->DR;                    // La lecture de DR efface le flag EOC
        adc_ready = 1;                         // Signal à la tâche de mise à jour
    }
}

/**
 * @brief Interruption EXTI du bouton (PC13) – priorité 2.
 * 
 * Détecte un front descendant (appui) et change le mode de signal.
 * Le LCD sera mis à jour via le drapeau lcd_update.
 */
void EXTI15_10_IRQHandler(void) {
    if (EXTI->PR & EXTI_PR_PR13) {
        mode++;
        if (mode >= NB_MODES) mode = 0;
        lcd_update = 1;                        // Demander un rafraîchissement
        EXTI->PR = EXTI_PR_PR13;               // Effacer le flag
    }
}

/**
 * @brief Interruption SysTick (1 kHz) – priorité basse (3).
 * 
 * Incrémente un compteur utilisé par les protothreads pour les temporisations.
 */
void SysTick_Handler(void) {
    sys_tick++;
}

/* ============================================================
 * 7. PROTOTHREADS (TÂCHES NON CRITIQUES)
 * ============================================================ */

/**
 * @brief Protothread de mise à jour de la fréquence.
 * 
 * Attend le sémaphore adc_ready (posé par l'interruption ADC).
 * Lorsqu'une nouvelle valeur est disponible, elle est filtrée par moyenne
 * exponentielle (poids 7/8), puis convertie en fréquence entre FREQ_MIN et FREQ_MAX.
 * Enfin, le pas de phase phase_step est recalculé et le LCD est mis à jour.
 * 
 * Ce protothread s'exécute en arrière-plan sans bloquer le système.
 */
PT_THREAD(task_update_frequency(struct pt *pt)) {
    static uint32_t filtered = 0;              // Valeur filtrée (persistante)

    PT_BEGIN(pt);

    while (1) {
        // Attendre qu'une nouvelle donnée ADC soit disponible (non-bloquant)
        PT_WAIT_UNTIL(pt, adc_ready == 1);
        adc_ready = 0;                         // Consommer le sémaphore

        // Filtrage exponentiel : on conserve 7/8 de l'ancienne valeur
        filtered = (filtered * 7 + adc_raw) >> 3;
        uint16_t adc_filt = filtered;

        // Conversion de la valeur ADC (0..4095) en fréquence (FREQ_MIN..FREQ_MAX)
        uint32_t freq = FREQ_MIN + (adc_filt * (FREQ_MAX - FREQ_MIN)) / 4095;
        if (freq < FREQ_MIN) freq = FREQ_MIN;
        if (freq > FREQ_MAX) freq = FREQ_MAX;
        freq_hz = (uint16_t)freq;

        // Calcul du pas de phase DDS : phase_step = (freq * 2^32) / SAMPLE_RATE
        // On utilise 4294967296ULL = 2^32 pour éviter les flottants
        phase_step = (uint32_t)((uint64_t)freq * 4294967296ULL / SAMPLE_RATE_HZ);

        // Demander la mise à jour du LCD
        lcd_update = 1;
    }

    PT_END(pt);
}

/**
 * @brief Protothread d'affichage LCD.
 * 
 * Rafraîchit l'écran périodiquement (toutes les 200 ms) ou sur demande (lcd_update).
 * Affiche le nom du mode, la fréquence et la valeur ADC brute.
 * 
 * Utilise la bibliothèque LCD fournie (LCD_PrintText) qui est bloquante, mais
 * comme elle n'est appelée qu'à faible fréquence, cela n'impacte pas le système.
 */
PT_THREAD(task_lcd(struct pt *pt)) {
    static uint32_t last_tick = 0;
    static char buffer[24];

    PT_BEGIN(pt);

    LCD_Clear();
    LCD_PrintText(0, 0, "Signal: Sinus");
    LCD_PrintText(0, 1, "Freq: 10 Hz");

    while (1) {
        // Attendre 200 ms (200 ticks à 1 kHz) ou une demande de mise à jour
        PT_WAIT_UNTIL(pt, (sys_tick - last_tick) >= 200 || lcd_update == 1);
        last_tick = sys_tick;
        lcd_update = 0;

        // Ligne 0 : nom du signal
        LCD_PrintText(0, 0, "Signal:");
        LCD_PrintText(8, 0, (char*)mode_names[mode]);

        // Ligne 1 : fréquence et valeur ADC
        sprintf(buffer, "Freq:%3dHz ADC:%4d", freq_hz, adc_raw);
        LCD_PrintText(0, 1, buffer);
    }

    PT_END(pt);
}

/* ============================================================
 * 8. FONCTION PRINCIPALE (main)
 * ============================================================ */

int main(void) {
    // 1. Initialisation du matériel
    clock_init();                            // Horloge HSI 16 MHz
    Delay_Init();                            // Initialisation des délais pour le LCD
    gpio_init();                             // GPIO (SPI, ADC, bouton)
    spi_init();                              // SPI pour le DAC MCP4921
    adc_timer_init();                        // ADC déclenché par TIM3 (100 Hz)
    tim2_init();                             // TIM2 pour le DDS (10 kHz)
    exti_init();                             // Bouton (PC13) en interruption

    // 2. Génération de la table de sinus
    generate_sine_table();

    // 3. Initialisation du LCD (bloquant mais une seule fois)
    LCD_Init();
    LCD_Clear();

    // 4. Valeur initiale de la fréquence (10 Hz)
    freq_hz = 10;
    phase_step = (uint32_t)((uint64_t)10 * 4294967296ULL / SAMPLE_RATE_HZ);
    phase_acc = 0;

    // 5. Configuration de SysTick pour les temporisations (1 kHz)
    SysTick_Config(16000000 / 1000);
    NVIC_SetPriority(SysTick_IRQn, 3);       // Priorité basse

    // 6. Initialisation des protothreads
    static struct pt pt_freq, pt_lcd;
    PT_INIT(&pt_freq);
    PT_INIT(&pt_lcd);

    // 7. Boucle principale : scheduler des protothreads
    while (1) {
        // Exécution coopérative des tâches
        PT_SCHEDULE(task_update_frequency(&pt_freq));
        PT_SCHEDULE(task_lcd(&pt_lcd));
        // D'autres protothreads pourraient être ajoutés ici
    }
}
